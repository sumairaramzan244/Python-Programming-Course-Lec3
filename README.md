## Lecture 3 — Variables (Advanced Concepts)

In Lecture 3, we went deeper into Variables in Python.
After learning the basics in Lecture 2, this lecture covered
how to properly name variables, work with multiple variables
at once, unpack lists, and understand the difference between
global and local variables.

---

## Topics Covered

**1. Casting & Type Checking**  
Converting data from one type to another and checking variable types.

**2. Naming Conventions**  
Three popular ways to name variables in Python:
* Camel Case  — myVariableName
* Pascal Case — MyVariableName  
* Snake Case  — my_variable_name

**3. Multiple Assignment**  
Assigning multiple variables in a single line of code.

**4. Unpacking**  
Extracting values from a list and storing them in separate variables directly.

**5. Output Variables**  
Storing the result of a computation in a variable for later use.

**6. Global vs Local Variables**  
* Global Variable — defined outside a function, accessible everywhere
* Local Variable  — defined inside a function, only accessible there
* global keyword  — used to modify a global variable from inside a function

---

## What I Learned

* Always follow a naming convention to keep your code clean and readable
* Multiple variables can be assigned in one line — x, y, z = "cat", "dog", "bird"
* A list can be unpacked directly into separate variables
* Plus ( + ) adds numbers and joins strings together
* A local variable inside a function does not affect the global variable
* The global keyword gives a function access to modify a global variable
